Performance I.

¿Cómo mejoramos la performance?

  1. Vamos a hacer una primera auditoría para ver como estamos. Usaremos la opción de Simulated Slow 4G, 4x de CPU slowddown.
  2. Usaremos una fuente de reemplazo, asi tenemos siempre el contenido visible hasta que la fuente se ha descargador.
    Usaremos la propiedad font-display: swap para tener una fuente de sustitución.
    Ahora hemos solucionado un problema bastante común (https://css-tricks.com/fout-foit-foft/).
  3. Si volvemos a pasar un test, vemos como aún podemos mejorar ya que Lighthouse nos avisa que tenemos ficheros sin minificar.
    Para solucionarlo, dentro del html tendríamos que cambiar script.js por script.min.js, y hacemos lo mismo con los css y pasamos de nuevo el audit.
  4. Aun con una página tan senzilla podemos optimizar incluso la imágen, usando el formato correcto para cada navegador.
    Si usamos Chrome, la mejor opción és webp, por lo que cambiamos la extensión de la imágen que cargamos y vemos como la imágen pesará menos pero no perdemos calidad.
